Working on your own machine

Setup & tools

Colab keeps working for everything on this course, so none of this is urgent. But the exam is written on your own laptop, in VS Code, with no internet, and that should not be the first time you use it. Session 3 opens by walking through the four steps below, so come with them done if you can.

needed you will use it on this course  ·  extra good to know, not required

The four steps needed

Do them in order. Each link is the official guide, not a blog post that may have rotted.

  1. Install Python. Download the latest release for your system. On Windows, tick Add python.exe to PATH in the installer: it is easy to miss and awkward to fix later. python.org/downloads
  2. Install VS Code. The editor itself. It is not Python, and installing one does not install the other. code.visualstudio.com/download
  3. Add the Python extension. This is what teaches the editor about Python: running code, completing names, showing documentation when you hover. VS Code Python tutorial · then notebooks in VS Code
  4. Install the packages. Open a new notebook and run this in a cell:
    %pip install numpy pandas matplotlib scikit-learn
    You do this once, ever. If you would rather use a terminal, pip install with the same names does exactly the same thing.
If a step fails, bring the error message. Not a description of it, the text itself. Nearly every setup problem is solved in a minute by someone who can read the actual message, and unsolvable by guessing.

Python, VS Code, and the extension are three things needed

The commonest week-one confusion is thinking VS Code is Python. It is not. Python is the language: the thing that actually runs your code. VS Code is the editor: where you type and read it. The Python extension is the bridge between them. Three separate installs that then work together.

Which Python is it using? needed

A machine can have several Pythons on it, and VS Code has to be told which one to use. The first time you open a notebook it will ask, and it remembers the answer. You can change it later from the kernel picker at the top right of the notebook, or with Ctrl+Shift+P then "Python: Select Interpreter".

Almost every "but I installed it!" problem is this: the package went into one Python and VS Code is running a different one. Using %pip install from inside the notebook avoids it entirely, because %pip installs into whichever Python that notebook is actually using.

Where your data file has to sit needed

In Colab you uploaded a file. Locally you just put it next to your code:

mlfin/
├── analysis.ipynb
└── data/
    └── prices.csv

Then pd.read_csv("data/prices.csv") finds it. That path is read relative to where you are running from, not relative to where the file lives, which is why the fix for FileNotFoundError is almost always the same: in VS Code, open the folder (File → Open Folder), not just the single file.

The course notebooks are more forgiving than that. They look in a few likely places and, failing all of them, download the data from this repository. So they work in Colab with nothing uploaded, and locally with the data/ folder beside them.

When something breaks needed

What you seeWhat it meansWhat to do
ModuleNotFoundError: No module named 'pandas' Python looked for the package and it is not there. Run %pip install pandas in a cell. If it was already installed, you are running a different Python: check the interpreter.
FileNotFoundError: 'data/prices.csv' The path is right or wrong relative to where you are running, and you are not where you think you are. Open the folder rather than the file. import os; os.getcwd() tells you where you actually are.
No kernel, or "select a kernel" and nothing sensible listed VS Code cannot find a Python to run the notebook with. Check Python installed correctly, then pick it from the kernel list. You may need %pip install jupyter once.
NameError: name 'prices' is not defined The cell that creates prices has not run in this session. Run the cells above, or Run All. Notebooks remember what you ran, not what is written above.
SettingWithCopyWarning pandas cannot tell whether you meant to change a selection or the original table. Add .copy() when you take the selection you are about to modify.
A cell runs forever Usually a while loop whose condition never becomes false. Press the stop button, then check that something inside the loop actually changes what the condition tests.

Scripts and notebooks: .py against .ipynb extra

This course lives in notebooks, and so does the exam, so you do not need scripts. But you will meet them, and the difference is worth a minute.

A notebook, .ipynbA script, .py
How it runsOne cell at a time, in whatever order you press RunTop to bottom, every time
OutputSits under each cell, and is saved in the filePrinted to a terminal, and gone
Good forExploring, and for anything with figures or a written argumentWork you will repeat, and code other files will import
WeaknessHidden state: what you ran can differ from what is writtenNo inline output, so exploring is clumsy

A common working pattern is to explore in a notebook, and move anything you have stopped changing into a .py file you can import. Both open in VS Code with no extra setup.

Virtual environments extra

You will read about these. The idea: instead of one shared set of packages for your whole machine, each project gets its own. That way upgrading a package for one project cannot quietly break another, and you can record exactly which versions a project needs.

For this course you do not need one. One Python with the four packages installed is completely fine. If your setup created a .venv folder, that is what it was: select it as your interpreter and carry on.

If you want to understand them: Python environments in VS Code.

What pip actually does extra

Python ships with a standard library: modules like math, random and statistics that are always there. Everything else lives on PyPI, a public archive of Python packages. pip is the program that fetches a package from PyPI and installs it into your Python, along with anything that package itself depends on.

So %pip install pandas is a download, once. import pandas is what makes it available in a particular notebook, every time. Installing packages is the official explanation.

VS Code, once it is working extra

  • Hover a name to see its documentation. This works offline, because the text ships inside the package.
  • Ctrl+Space inside brackets lists the arguments a function takes.
  • F12 jumps into the package's own source code, which is more readable than you would expect.
  • Ctrl+Shift+P opens the command palette: everything VS Code can do, searchable by name. If you only learn one shortcut, learn this.
  • Shift+Enter runs the current cell and moves on.

Git and GitHub extra

Not used on this course, and not needed for the exam. Worth knowing about anyway, because it is how essentially all code is kept: a full history of every change, the ability to go back to any earlier version, and a way for several people to work on the same files without overwriting each other. This site and all its materials live in a Git repository.

If you are curious: GitHub's own introduction and git-scm.com. VS Code has Git support built in, so you would not need a separate program.
