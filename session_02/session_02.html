The accumulator pattern

That shape comes up constantly:

  • start a variable at a neutral value (0 for a sum)
  • update it once per item, inside the loop
  • read it after the loop

Set up, then loop and update, then use the result. Summing, counting, and averaging all follow this one pattern.

A shorthand: +=

Updating a variable from itself, total = total + r, is so common that Python gives it a shorthand:

total += r means total = total + r. The same works for -=, *=, and /=. We will use it for accumulators from here on.

3 · While loops

Part 3

Repeating for as long as a condition stays true.

When you do not know how many times

A for loop runs once per item. A while loop runs for as long as a condition holds, however many times that takes.

How many years to grow 100 into 150 at 10% a year?

You did not have to know the answer in advance. The loop keeps going until value reaches 150.

Careful: it must be able to stop

Something inside the loop has to move the condition towards False, or it runs forever:

value = 100
while value < 150:
    years += 1     # value never changes, so this never stops

A while loop needs an exit. Make sure each pass changes something the condition depends on. If a cell seems stuck, interrupt it with the stop button.

4 · Your own functions

Part 4

Naming a calculation once, so you can reuse it anywhere.

You already use functions

print, len, max, and sum are functions: a name, then inputs in brackets. Today you write your own.

Defining a function

def creates a function. You give it a name, its inputs, and what it returns:

The shape of a def

def simple_return(p0, p1):     # name, and inputs (the parameters)
    return (p1 - p0) / p0      # the value it hands back
  • def name(parameters): opens the function
  • the indented body is what it does
  • return hands a value back to whoever called it

Call it as often as you like

Once defined, a function can be used again and again:

Write the formula once, and trust it everywhere. There is no copy to get wrong.

Parameters and arguments

The names in the def are parameters; the values you pass in are arguments:

def simple_return(p0, p1):   # p0 and p1 are parameters
    return (p1 - p0) / p0

simple_return(100, 110)      # 100 and 110 are arguments

Inside that call, p0 is 100 and p1 is 110.

return hands a value back

Without return, a function does its work but gives nothing back:

x is None: show printed a value but did not return one. Use return when you want to keep the result.

A parameter can have a default, used when you do not pass it:

6 · Measuring risk properly

Part 6

Everything from today, combined into a real measure of risk.

From range to volatility

Last session’s risk gauge used only two days, the highest and the lowest. A better measure uses every day’s return. That measure is volatility: how much the daily returns typically spread around their average.

What volatility is

Volatility is the standard deviation of the daily returns:

\[ \sigma = \sqrt{\ \frac{1}{n}\sum_{i} \left(r_i - \bar{r}\right)^2\ } \]

In words: take each return’s distance from the average, square it, average those squares, and take the square root. It is the typical size of a daily move.

The calculation, in three steps

Each step is something you can already do:

  1. the daily returns, with a loop (Part 1)
  2. their average, sum(returns) / len(returns)
  3. each return’s squared distance from the average, averaged, then square-rooted with ** 0.5

We now build these into one function, one step at a time.

Build it up: the daily returns

Start the function with just step 1, the returns loop. It is not finished yet:

def volatility(prices):
    returns = []
    for i in range(1, len(prices)):
        returns.append((prices[i] - prices[i - 1]) / prices[i - 1])
    # still to come: the average, and the spread

So far it only collects the daily returns into a list.

Add step 2, the average return. Still not finished:

def volatility(prices):
    returns = []
    for i in range(1, len(prices)):
        returns.append((prices[i] - prices[i - 1]) / prices[i - 1])
    mean = sum(returns) / len(returns)
